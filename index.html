How research thinking becomes project delivery.

The same pattern runs through research, engineering, and client work: clarify the problem, map constraints, build a usable artifact, document the result.

Frame

Define the problem, users, constraints, and success criteria before choosing a tool or build path.

Output: scope brief

Design

Translate the concept into architecture, workflow, curriculum, prototype plan, or page structure.

Output: project plan

Build

Create the visible artifact: website, workflow, demo, report, experiment package, or training module.

Output: working artifact

Deliver

Document decisions, prepare handoff material, surface risks, and make the next step clear.

Output: reviewable handoff
